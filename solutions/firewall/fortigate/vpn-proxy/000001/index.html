Blocking Known VPNs
& Evasive Proxies

Modern VPNs use SSL masking, QUIC encapsulation, and rotating cloud IPs to bypass traditional firewalls. This guide provides a zero-trust architecture approach to identify and block these tunnels in FortiOS 7.6.

Start Implementation Quick Checklist
1

1.The Internet Service Database (ISDB)

The ISDB is your first line of defense. It blocks traffic based on destination IP ownership. This is high-performance because it triggers at the routing/policy layer before the traffic is even scanned.

VPN-Anonymous

Commercial VPNs like Nord, Express, and Surfshark.

Proxy-Proxy.Server

Public web-proxies, SOCKS5, and anonymizers.

Tor-Relay.Node

Known entrance and exit nodes for the Tor network.

CLI Shortcut:

config firewall policy
  edit 0
    set name "DENY_KNOWN_VPN_ISDB"
    set internet-service enable
    set internet-service-name "VPN-Anonymous.VPN" "Proxy-Proxy.Server" "Tor-Relay.Node"
    set action deny
  next
end

2.Application Control Signatures

Application Control looks for behavioral patterns. Even if a user hosts a private VPN on their home PC, the FortiGate can recognize the "OpenVPN" or "WireGuard" handshake.

Configuration Tip

Don't just block the "Proxy" category. Go to Filter Overrides and block specific high-risk apps like Psiphon, Ultrasurf, and Hotspot Shield individually to ensure higher accuracy.

GUI Navigation: Security Profiles > Application Control

Proxy Category BLOCK
Tunneling Behavior BLOCK
Unknown Applications MONITOR
3

3.Deep SSL Inspection

Without this, your firewall is blind. Modern VPNs hide inside HTTPS (Port 443). Deep Inspection decrypts the session so the Application Control engine can see the tunnel attempt.

The "Blinder" Effect

SSL Inspection (Certificate Inspection only) can only see the SNI (e.g., google.com). A VPN can easily spoof this or use "Domain Fronting".

The Solution

Deploy Deep-Inspection. This requires the FortiGate CA Certificate to be installed on all client machines via GPO or MDM.

"Inspect what you expect"

Required for Layer 2 and Layer 4 to be 100% effective.

4

4.Blocking the QUIC Protocol

QUIC (UDP 443) is used by browser-based VPN extensions and Google services. Because it's encrypted and faster than TCP, it is the primary method used to bypass web filters.

Why Block QUIC?

  • Forces browser fallback to standard TLS (Port 443).
  • Standard TLS can be inspected by the SSL Engine.
  • Prevents "shadow tunnels" inside UDP streams.
5

5.Policy Hierarchy & Logic

Firewall policies are processed from the top down. If your VPN block rule is below your "General Internet" rule, it will never trigger.

ID Policy Name Destination Action
1 DENY_VPN_AND_PROXY ISDB: VPN-Anonymous DENY
2 ALLOW_GENERAL_INTERNET All (WAN) ACCEPT
