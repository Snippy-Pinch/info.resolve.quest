🛡️ VPN & Proxy Blocking

Blocking Known VPNs & Evasive Proxies

Modern VPNs use SSL masking, QUIC encapsulation, and rotating cloud IPs to bypass traditional firewalls. This guide provides a zero-trust architecture approach for FortiOS 7.6.

🕳️
The Problem

Evasion Techniques

VPNs and proxies are designed to hide. They use encrypted tunnels, domain fronting, and cloud infrastructure to look like normal HTTPS traffic.

⚡
SSL Masking

VPN traffic hides inside standard HTTPS on Port 443.

⚡
QUIC Tunnels

Browser-based VPNs use encrypted UDP to bypass TCP inspection entirely.

⚡
Rotating IPs

Cloud-based VPNs constantly change IP addresses to avoid blacklists.

🛡️
The Solution

5-Layer Defense

No single layer catches everything. By stacking five complementary techniques, you create a defense-in-depth that covers each evasion method.

The Internet Service Database (ISDB)

Your first line of defense. Blocks traffic based on destination IP ownership. High-performance because it triggers at the routing/policy layer before traffic is even scanned.

VPN-Anonymous

Nord, Express, Surfshark, etc.

Proxy-Proxy.Server

Web-proxies, SOCKS5, anonymizers.

Tor-Relay.Node

Known Tor entrance & exit nodes.

CLI Configuration:

config firewall policy
  edit 0
    set name "DENY_KNOWN_VPN_ISDB"
    set internet-service enable
    set internet-service-name "VPN-Anonymous.VPN" "Proxy-Proxy.Server" "Tor-Relay.Node"
    set action deny
  next
end

Application Control Signatures

Looks for behavioral patterns. Even if a user hosts a private VPN on their home PC, the FortiGate can recognize the "OpenVPN" or "WireGuard" handshake.

Configuration Tip

Don't just block the "Proxy" category. Go to Filter Overrides and block specific high-risk apps like Psiphon, Ultrasurf, and Hotspot Shield individually.

Security Profiles > Application Control
Proxy Category BLOCK
Tunneling Behavior BLOCK
Unknown Applications MONITOR

Deep SSL Inspection

Without this, your firewall is blind. Modern VPNs hide inside HTTPS (Port 443). Deep Inspection decrypts the session so the Application Control engine can see the tunnel attempt.

The "Blinder" Effect

Certificate Inspection only sees the SNI (e.g., google.com). A VPN can spoof this or use "Domain Fronting" to look like legitimate traffic.

The Solution

Deploy Deep-Inspection. Requires the FortiGate CA Certificate on all client machines via GPO or MDM.

"Inspect what you expect"

Required for Layer 2 and 4 to be 100% effective.

Blocking the QUIC Protocol

QUIC (UDP 443) is used by browser-based VPN extensions and Google services. Because it's encrypted and faster than TCP, it's the primary method used to bypass web filters.

Why Block QUIC?

  • Forces TLS fallback. Browser reverts to standard HTTPS on Port 443.
  • Enables inspection. Standard TLS can be decrypted by the SSL engine.
  • Eliminates shadow tunnels. Prevents covert channels inside UDP streams.

Policy Hierarchy & Logic

Firewall policies are processed top-down. If your VPN block rule sits below your "General Internet" rule, it will never trigger.

ID Policy Name Destination Action
1 DENY_VPN_AND_PROXY ISDB: VPN-Anonymous DENY
2 ALLOW_GENERAL_INTERNET All (WAN) ACCEPT

⚠️ Critical Rule

The DENY_VPN_AND_PROXY rule must always be above your general allow rules. The FortiGate processes policies sequentially—first match wins. If the allow rule matches first, the deny rule is never evaluated.
